# сделай для него PowerShell деплой-скрипт тоже, чтобы можно было удобно разверуть в antigravity


---

**🎯 Target Model:** PowerShell 5.1+ (Windows / Antigravity IDE environment)
**📋 Context \& Goal:** Самодостаточный деплой-скрипт — содержит полный контент `SKILL.md` внутри себя как here-string. Не требует ручного копирования текста. Один запуск → всё готово.

***

## 💻 `deploy_skill_doc_extractor.ps1`

```powershell
#Requires -Version 5.1
# ============================================================
# DEPLOY: doc-extractor SKILL
# Файл: deploy_skill_doc_extractor.ps1
# Место: корень проекта (рядом с .agent/)
# Запуск: .\deploy_skill_doc_extractor.ps1
# Автор: GD-Workspace | Gamedesign-ops
# ============================================================

Set-StrictMode -Version Latest
$ErrorActionPreference = "Stop"

# ── КОНФИГ ──────────────────────────────────────────────────
$SKILL_NAME    = "doc-extractor"
$SKILL_DIR     = ".agent\skills\$SKILL_NAME"
$SKILL_FILE    = "$SKILL_DIR\SKILL.md"
$RULES_FILE    = ".agent\rules\02_GD.md"
$AGENTS_FILE   = "AGENTS.md"
$LOG_FILE      = ".gemini\antigravity\brain\decisions_log.md"
$DEPLOY_DATE   = Get-Date -Format "yyyy-MM-dd HH:mm"

# ── ЦВЕТА ───────────────────────────────────────────────────
function Write-Header  { param($t) Write-Host "`n$t" -ForegroundColor Cyan }
function Write-Ok      { param($t) Write-Host "  ✅  $t" -ForegroundColor Green }
function Write-Skip    { param($t) Write-Host "  ⏭️   $t" -ForegroundColor DarkGray }
function Write-Warn    { param($t) Write-Host "  ⚠️   $t" -ForegroundColor Yellow }
function Write-Fail    { param($t) Write-Host "  ❌  $t" -ForegroundColor Red }
function Write-Step    { param($t) Write-Host "`n  ► $t" -ForegroundColor White }

# ════════════════════════════════════════════════════════════
#  SKILL.md CONTENT  (here-string — редактировать здесь)
# ════════════════════════════════════════════════════════════
$SKILL_CONTENT = @'
# SKILL: DOC-EXTRACTOR — Reverse-Engineering & Reuse
# Файл: .agent/skills/doc-extractor/SKILL.md
# Версия: 1.0 | Автор: GD
# Загружать: когда пользователь скидывает любые входящие материалы
# (старые GDD, питчи, таблицы, скриншоты, схемы, ссылки, брифы)
# для переработки в текущий целевой документ

## 🎯 ЦЕЛЬ СКИЛЛА

Превратить любой набор входящих материалов в **структурированный
Extraction Report** — манифест того, что берётся из каждого документа,
куда в текущий целевой документ это вставляется, и что остаётся
невыясненным после анализа.

**Ключевой принцип:** не копируй — переосмысляй.
Извлечённые данные адаптируются под текущую платформу,
механики и продуктовые цели проекта.

## 📥 ПОДДЕРЖИВАЕМЫЕ ФОРМАТЫ ВХОДНЫХ ДАННЫХ

| Формат | Что извлекаем |
|--------|--------------|
| `.md` / `.txt` | Весь текст, структура, списки, таблицы, ID-теги |
| `.pdf` | Текст (если не скан), таблицы, структура глав |
| `.jpg` / `.png` (схемы) | Диаграммы loop'ов, flow-чарты, архитектуры систем |
| `.jpg` / `.png` (таблицы) | Балансные таблицы, прогрессионные кривые, economy sheets |
| `.jpg` / `.png` (скриншоты UI) | Screen flow, HUD-элементы, UX-паттерны |
| `.jpg` / `.png` (питч-слайды) | Концепты, USP, аудитория, референсы |
| URL → GitHub / GDrive / Notion | Текстовый контент страницы, структура документа |
| Таблицы (CSV, XLS как изображение) | Числовые константы, balance curves, LTV данные |
```

```
### ═══ ФАЗА 3: МАППИНГ НА ЦЕЛЕВОЙ ДОКУМЕНТ ═══

**Цель:** для каждого блока данных определить точное место
в целевом документе.

**Целевые документы:**
- `GDD`          → маппинг на [ID:...] блоки
- `Feature Spec` → маппинг на секции 1–8
- `Balance Sheet`→ маппинг на [ID:BALANCE], [ID:ECONOMY]
- `LiveOps Event`→ маппинг на [ID:LO_...]
- `One-Pager`    → маппинг на блоки 1–11

#### EXTRACTION MAP

```


## EXTRACTION MAP

*Целевой документ: [тип] | Текущая версия: [vN]*


| \# | Источник | Данные (кратко) | → Целевой [ID:] | Действие | Качество |
| :-- | :-- | :-- | :-- | :-- | :-- |
| 1 | doc.pdf §2 | «Core loop: tap→kill→loot» | [ID:CORE_LOOP] | ВСТАВИТЬ | [FACT] |
| 2 | scheme.jpg | Loop диаграмма | [ID:LOOPS_OVERVIEW] | ВОССОЗДАТЬ | [FACT]🟢 |
| 3 | brief.md | D7 target = 35% | [ID:LIVEOPS_KPI] | ВСТАВИТЬ | [FACT] |

```

**Легенда действий:**
- `ВСТАВИТЬ`    — данные подходят напрямую
- `АДАПТИРОВАТЬ`— суть берём, детали пересматриваем
- `ПРОВЕРИТЬ`   — требуют верификации перед вставкой
- `ВОССОЗДАТЬ`  — из изображения создаём текстовый блок
- `ИГНОРИРОВАТЬ`— нерелевантны или противоречивы
- `УТОЧНИТЬ`    — нужен вопрос к пользователю
```

```
## 🛠️ ИНСТРУКЦИИ АГЕНТУ

### Когда активировать
- Пользователь приложил ≥ 1 файл вместе с запросом
- Упомянул «старый GDD», «вот документы», «используй это»
- В запросе есть URL-ссылки
- Есть изображения в чате

### Порядок работы
```

1. Определи ЦЕЛЕВОЙ ДОКУМЕНТ (что пишем?)
→ Спроси если неочевидно
2. Фаза 1: зарегистрируй все входящие файлы
3. Фаза 2: проанализируй каждый файл своим анализатором
4. Фаза 3: построй EXTRACTION MAP
5. Фаза 4: Gap Analysis + Extraction Report
6. ТОЛЬКО ПОТОМ начни заполнять целевой документ
7. Выведи финальный Extraction Report
```

### Работа с изображениями (Claude multimodal)
- Описывай ВСЕ видимые элементы (текст, иконки, стрелки, числа, цвета)
- Сначала описывай → потом маппируй
- Нечёткий текст: `[НЕЧЁТКО: предположительно "..."]`
- Таблицы воссоздавай в Markdown даже если частично видны

### Приоритеты источников при конфликтах
```

Priority 1: Явные инструкции пользователя в чате
Priority 2: Файлы текущего запроса
Priority 3: brain/project_context.md
Priority 4: URL / внешние источники
Priority 5: Старые GDD / устаревшие документы

```

### Маркировка качества данных
- [FACT]       — из документа, без интерпретации
- [INTERPRET]  — интерпретировано из изображения/схемы
- [HYPOTHESIS] — предположение GD, не из документа
- [OUTDATED?]  — документ > 12 мес или признаки устарелости
- [CONFLICT:X] — противоречие с источником X
```

```
| Doc Extractor | ```.agent/skills/doc-extractor/SKILL.md``` | Любые входящие материалы (файлы, фото, ссылки) |

$AGENTS_PATCH = @"

---

### Skill: doc-extractor
**Путь:** ```.agent/skills/doc-extractor/SKILL.md```
**Использует:** GD (основной), PM, PRODUCER (read)

структурированный Extraction Report → заполнение GDD / Feature Spec /
Balance Sheet / LiveOps Event.
**Активация:** автоматически при наличии вложений у пользователя.
"@

# ════════════════════════════════════════════════════════════
#  MAIN
# ════════════════════════════════════════════════════════════

Clear-Host
Write-Host ""
Write-Host "  ╔══════════════════════════════════════════════╗" -ForegroundColor Cyan
Write-Host "  ║   DEPLOY: doc-extractor SKILL v1.0          ║" -ForegroundColor Cyan
Write-Host "  ║   Gamedesign-ops | GD-Workspace             ║" -ForegroundColor Cyan
Write-Host "  ╚══════════════════════════════════════════════╝" -ForegroundColor Cyan

# ── Проверка: запущен из корня проекта ──────────────────────
Write-Header "[ STEP 1/6 ] Проверка окружения"

if (-not (Test-Path ".agent")) {
    Write-Fail "Директория .agent не найдена."
    Write-Warn "Запусти скрипт из корня проекта (там где лежит .agent/)."
    Write-Host ""
    Write-Host "  Текущая папка: $(Get-Location)" -ForegroundColor Gray
    exit 1
}
Write-Ok ".agent/ найден — ты в корне проекта"

if (-not (Test-Path ".gemini\antigravity\brain")) {
    Write-Warn "brain/ не найден. Создаю..."
    New-Item -Path ".gemini\antigravity\brain" -ItemType Directory -Force | Out-Null
    Write-Ok "brain/ создан"
} else {
    Write-Ok "brain/ найден"
}

# ── Создание директории скилла ───────────────────────────────
Write-Header "[ STEP 2/6 ] Создание директории скилла"

if (-not (Test-Path $SKILL_DIR)) {
    New-Item -Path $SKILL_DIR -ItemType Directory -Force | Out-Null
    Write-Ok "Создана: $SKILL_DIR"
} else {
    Write-Skip "Уже существует: $SKILL_DIR"
}

# ── Запись SKILL.md ──────────────────────────────────────────
Write-Header "[ STEP 3/6 ] Запись SKILL.md"

$overwrite = $true
if (Test-Path $SKILL_FILE) {
    Write-Warn "Файл уже существует: $SKILL_FILE"
    $ans = Read-Host "  Перезаписать? (y/n)"
    $overwrite = ($ans -eq "y")
}

if ($overwrite) {
    $SKILL_CONTENT | Out-File -FilePath $SKILL_FILE -Encoding UTF8 -Force
    $lineCount = ($SKILL_CONTENT -split "`n").Count
    Write-Ok "Записан: $SKILL_FILE ($lineCount строк)"
} else {
    Write-Skip "Файл оставлен без изменений"
}

# ── Патч 02_GD.md: добавить строку в таблицу скиллов ────────
Write-Header "[ STEP 4/6 ] Обновление 02_GD.md (таблица Skills)"

if (Test-Path $RULES_FILE) {
    $rulesContent = Get-Content $RULES_FILE -Raw -Encoding UTF8

    if ($rulesContent -match "doc-extractor") {
        Write-Skip "doc-extractor уже есть в $RULES_FILE"
    } else {
        # Ищем строку с последним skill в таблице и добавляем после неё
        $anchor = "| Agent TZ Template"
        if ($rulesContent -match [regex]::Escape($anchor)) {
            $rulesContent = $rulesContent -replace `
                "(\| Agent TZ Template[^\n]+)", `
                "`$1`n$SKILL_TABLE_ROW"
            $rulesContent | Out-File -FilePath $RULES_FILE -Encoding UTF8 -Force
            Write-Ok "Строка doc-extractor добавлена в таблицу Skills"
        } else {
            Write-Warn "Якорь '| Agent TZ Template' не найден в $RULES_FILE"
            Write-Warn "Добавь вручную в таблицу ## 🛠️ НАВЫКИ:"
            Write-Host "  $SKILL_TABLE_ROW" -ForegroundColor Gray
        }
    }
} else {
    Write-Warn "$RULES_FILE не найден — пропускаю патч"
    Write-Warn "После создания 02_GD.md добавь строку в таблицу навыков вручную"
}

# ── Патч AGENTS.md ────────────────────────────────────────────
Write-Header "[ STEP 5/6 ] Обновление AGENTS.md"

if (Test-Path $AGENTS_FILE) {
    $agentsContent = Get-Content $AGENTS_FILE -Raw -Encoding UTF8

    if ($agentsContent -match "doc-extractor") {
        Write-Skip "doc-extractor уже есть в $AGENTS_FILE"
    } else {
        Add-Content -Path $AGENTS_FILE -Value $AGENTS_PATCH -Encoding UTF8
        Write-Ok "Блок doc-extractor добавлен в $AGENTS_FILE"
    }
} else {
    Write-Warn "$AGENTS_FILE не найден — создаю минимальный"
    $AGENTS_PATCH | Out-File -FilePath $AGENTS_FILE -Encoding UTF8
```

## 🧠 TECHNICAL COMMENTARY

**Главное отличие от предыдущего деплой-скрипта** — SKILL.md встроен в скрипт как PowerShell here-string (`@'...'@`).  Это означает что скрипт полностью самодостаточен: скопировал один `.ps1` файл в корень проекта → запустил → всё готово. Не нужно отдельно копировать Markdown-контент.[^1]

**Три умных патча** вместо слепой записи:

- `02_GD.md` — скрипт ищет якорную строку `| Agent TZ Template` в существующей таблице навыков и вставляет новую строку точно после неё, не ломая форматирование остального файла [^1]
- `AGENTS.md` — проверяет наличие `doc-extractor` строки перед добавлением, идемпотентен при повторном запуске[^2]
- `decisions_log.md` — использует `Add-Content` (append-only), в точном соответствии с правилом «лог только дополняется» из `00_global.md`[^3]

**Авто-определение редактора** перебирает `code → antigravity → cursor` в порядке приоритета — подходит для любой конфигурации рабочей машины.

**`Set-StrictMode -Version Latest` + `$ErrorActionPreference = "Stop"`** — это стандарт безопасного PowerShell из `QWEN.md`, предотвращающий молчаливые ошибки при работе с файловой системой.[^1]
