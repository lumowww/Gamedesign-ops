# UIUX-S1-001 — AiW: Игровые экраны Sprint 1
[STATUS:DRAFT] | Агент: UIUX | Sprint 1
Дата: Sprint 0 → Sprint 1

---

## 1. Context
**Project:** Alien in Waders (AiW)
**Related GDD blocks:** [ID:LOOPS_OVERVIEW]
**Problem:** Нет UI. Нужен минимальный набор экранов для тестового билда.
Стиль: функциональный, читаемый, mobile-first. Полноценный дизайн — Sprint 2+.

---

## 2. Goal & Success Criteria
**Goal:** Игрок может начать матч, управлять башнями, понять результат — без объяснений.
**KPI:** Все touch targets ≥ 48dp, 0 элементов перекрывающих игровое поле в центре экрана.

---

## 3. Scope — 5 Экранов Sprint 1

---

### Экран 1: Главное меню (MainMenu)

**Элементы:**
- Кнопка **PLAY** — большая, центр экрана, основное действие
- Логотип AiW — placeholder (текст «ALIEN IN WADERS» сверху)
- Иконки-заглушки: Settings (шестерня) + Profile (силуэт) — не работают в Sprint 1

**Требования:**
- Кнопка PLAY = минимум 120×60dp
- Без анимаций при первом открытии (Sprint 2+)

---

### Экран 2: Игровой HUD (GamePlay)

**Элементы:**

**TOP-LEFT:**
- Score — крупное число ≥ 28pt, белый текст, тёмная обводка или подложка
- Обновляется в реальном времени при каждом убийстве крипа

**TOP-CENTER:**
- Wave indicator: «Волна 3 / 10» — средний размер, ≥ 20pt
- При новой волне — краткая анимация появления (0.3 сек)

**TOP-RIGHT:**
- Кнопка PAUSE — иконка ⏸, ≥ 48×48dp

**BOTTOM (нижняя 1/5 экрана — thumb reach zone):**
- Кнопка **Spawn: Огнемёт** — иконка башни + стоимость в очках под иконкой
- Кнопка **Spawn: Крио** — иконка башни + стоимость в очках
- Состояния кнопок:
  - ACTIVE: иконка яркая + цвет кнопки = цвет башни (оранжевый / синий)
  - DISABLED: иконка серая, кнопка серая — если очков недостаточно
  - Переход ACTIVE ↔ DISABLED мгновенный (без задержки)

**HIGHLIGHT (выделение башен одного типа):**
- При нажатии на башню на поле: все башни того же ТИПА И УРОВНЯ подсвечиваются (glow / outline)
- Это подсказка «эти можно слить»
- Подсветка гасится при нажатии в другое место

**MERGE PROMPT:**
- При наведении drag на подсвеченную башню: целевой слот подсвечивается иначе (пульсирует)

**Требования:**
- ❌ Никаких попапов / реклам / FTUE-оверлеев в первые 2 минуты игры
- ❌ UI не перекрывает центральную зону поля (башни и крипы должны быть видны)
- Score и Wave indicator не перекрываются между собой

---

### Экран 3: Win Screen (WinScreen)

**Элементы:**
- Заголовок: «ПОБЕДА!» — крупно, центр
  (или «ВОЛНА ПРОЙДЕНА!» если это промежуточная победа — уточнить с GD)
- Итоговый score — большое число
- Личный рекорд: «🏆 Новый рекорд!» — если побит (badge)
- Кнопка **ЕЩЁ РАЗ** (primary) — перезапуск матча
- Кнопка **МЕНЮ** (secondary) — возврат в MainMenu

**Требования:**
- Обе кнопки ≥ 120×56dp
- Небольшая анимация появления экрана (scale-in 0.3 сек) — DOTween

---

### Экран 4: Lose Screen (LoseScreen)

**Элементы:**
- Заголовок: **«Крип прорвался на волне N»** — КОНКРЕТНЫЙ текст, не «Поражение»
  (Readable Failure rule — игрок должен понять ЧТО произошло)
- Число прожитых волн
- Лучший личный результат
- Кнопка **ПОПРОБОВАТЬ СНОВА** (primary)
- Кнопка **МЕНЮ** (secondary)

**Требования:**
- ❌ Никаких агрессивных IAP-офферов на экране поражения в Sprint 1
- Номер волны = обязательный элемент (не опционально)

---

### Экран 5: Pause (PauseOverlay)

**Элементы:**
- Затемнённый полупрозрачный overlay (α ≈ 0.7, цвет #000)
- Надпись «ПАУЗА» — центр
- Кнопка **ПРОДОЛЖИТЬ** (primary)
- Кнопка **СДАТЬСЯ** (secondary, деструктивное действие — цвет предупреждения)

**Требования:**
- Overlay не скрывает геймплей полностью (игрок видит поле под ним)
- Кнопки ≥ 120×56dp

---

## 4. Общие UX Требования (ALL SCREENS)

- **Portrait mode ONLY** во всём Sprint 1
- **Touch target ≥ 48dp** для всех интерактивных элементов
- **Цветовой код башен проходит через UI:** цвет кнопки Spawn = цвет башни на поле
- **Шрифт:** sans-serif bold для чисел и заголовков, regular для подписей
- **Контраст:** текст поверх игрового поля — белый с обводкой или плашка
- **Без сложных анимаций** в Sprint 1 (только DOTween scale/fade, ≤ 0.5 сек)

**Референс стиля UI:**
- Функциональный sci-fi mobile: Kingdom Rush HUD approach (чистый, без лишнего шума)
- Не перегружать: игровое поле = главное, UI = инструмент

---

## 5. Acceptance Criteria

- [ ] Все 5 экранов реализованы в Unity Canvas
- [ ] Score обновляется в реальном времени без lag
- [ ] Spawn кнопки корректно переходят в disabled state при нехватке очков
- [ ] Lose screen показывает номер волны, на которой прорвался крип
- [ ] Все touch targets ≥ 48dp (проверить в Device Simulator)
- [ ] UI корректно отображается на 5" (1280×720) и 6.5" (2400×1080)
- [ ] Highlight башен работает при нажатии на башню на поле
- [ ] Pause overlay не мешает видеть поле (полупрозрачный)
- [ ] Кнопка СДАТЬСЯ выглядит деструктивно (отличается от ПРОДОЛЖИТЬ)
